写代码容易，写文档难。很多时候我们面对空白的 README.md 不知从何下笔，或者写出来的文档晦涩难懂，没人愿意看。  
  
偶然在 GitHub 发现 Awesome Technical Writing 这份资源合集，系统性地整理了技术写作相关的工具、教程和职业发展路径。  
  
收录了 Google、Microsoft 等大厂的标准风格指南，帮我们规范术语和格式。同时涵盖了从内容策划、信息架构到文档发布的完整工作流资源。  
  
GitHub：github.com/BolajiAyodeji/awesome-technical-writing  
  
甚至还整理了 Google Season of Docs 等开源文档贡献计划，以及静态站点生成器、语法检查器等辅助工具清单。  
  
无论你是想优化项目文档的开发者，还是计划转型专职技术写作，这份清单都值得收藏备用。
